**🔗 Live:** [portfolio-w5p8.vercel.app](https://portfolio-w5p8.vercel.app)

> **Headline:** Transforming Data Into Intelligent Business Decisions

## ✨ Features

- **Cinematic hero** with a live, canvas-based "data stream" / neural-network background and floating dashboard cards
- **Live GitHub integration** — repositories, languages, stars and follower counts are fetched directly from [github.com/SonuDeo](https://github.com/SonuDeo) and revalidated hourly (ISR)
- **Case-study project storytelling** — Challenge → Solution → Architecture → Results → Lessons
- **"Why Recruiters Hire Me"** dashboard with animated, count-up metrics
- **Interactive skills**, **experience timeline**, and **certification gallery**
- **Light (editorial) + dark (premium) themes** with a toggle
- **Full SEO**: metadata, Open Graph (dynamic OG image), JSON-LD `Person` schema, `robots.txt`, `sitemap.xml`, web manifest
- **Accessibility & performance first**: semantic HTML, skip link, focus states, `prefers-reduced-motion` support, lazy/paused animations

## 🖼️ Add your photo & resume

This repo ships with generated placeholders so it builds out of the box. Replace them with your real assets — **no code changes needed**:

| File | Replace with |
| --- | --- |
| `public/sonu.jpeg` | Your hero portrait (portrait orientation, ~900×1200 recommended) |
| `public/Sonu_Kumar_Resume.pdf` | Your resume PDF |

The placeholders are regenerated by the helper scripts in `scripts/` (require `Pillow` and `reportlab`).

## 🔧 Configuration

All personal content lives in **`src/lib/data.ts`** — name, role, links, skills, projects, certifications and timeline. Edit there to update copy anywhere on the site.

### Optional GitHub token

Public GitHub data works with no setup. To raise the API rate limit during frequent rebuilds, copy `.env.example` to `.env.local` and set:

```
GITHUB_TOKEN=ghp_xxx
```

## ☁️ Deploy to Vercel

1. Push this repo to GitHub.
2. Go to [vercel.com/new](https://vercel.com/new) and import the repository.
3. Framework preset is auto-detected as **Next.js** — no extra config required.
4. (Optional) Add `GITHUB_TOKEN` under **Settings → Environment Variables**.
5. Deploy. Update `site.url` in `src/lib/data.ts` to your final domain so OG tags and the sitemap use the correct absolute URLs.

Or via CLI:

```bash
npm i -g vercel
vercel        # preview
vercel --prod # production
```
